1. Пакет взаимодейсвтия с драйвером сервоприводов (servo):
    - отправка в ком-порт (send_com) данных из топика servo/to_write
    - получение из ком-порта (receive_com) и публикация в топик servo/current

2. Пакет взаимодейсвтия с камерами (camera):
    - получение потока с бинокулярной камеры (binocular_cam) и публикация в топик cam/binocular
    - получение потока с камеры на манипултяоре (arm_cam) и публикация в топик cam/arm

3. Пакет взаимодейсвтия с колёсами (wheel):
    - получение текущих об/мин с каждого колеса и публикация в топик wheel/current_rpm
    - чтение из топика wheel/target_rpm (об/мин) и отправка в ком-порт
        (получение и чтение данных реализовано в одном узле data_exchange - используется один uart канал)
            - для линейных и угловыйх скоростей используем стандратным протокол Twist,
            для обычных колёс имеем целевую линейную скрость и угловую по вепртикальной оси,
            которая представлет собой просто результат ПИДа по текущей ошибке
            - вычисление целевых rpm происходит по треубемой угловой скорости и ошибке по rpm в отдельном узле
            - вычисление угловой скорости происходит в отедльном узле по линейной ошибке - фактически по результату ПИДа от линейной ошибки

4. Пакет для rplidar от разарботчиков (sllidar_ros2):
    - получение данных от лидара (rplidar c1) и публикация их в топик /scan

5. Пакет обработки данных лидара (lidar):
    - чтение данных из топика /scan
    - формирование моментальной карты, обработка зон интереса, публикация в топик lidar 
        (get_main_obstacles публикует медианные расстояния по заданному размеру сектора окружности)

6. Пакет для работы моделей ML (coral):
    - a hihuya tut ne budet, huy nastriosh ebuchiy coral

7. Пакет для ручного управления роботом (keyboard):
    - соотвутствующие узлы формируют уставки для колёс и целевые обобщённые координаты для камеры и маниулятора соответственно
